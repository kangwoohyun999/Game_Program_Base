### 1. 유니티 인터페이스 및 기본 조작
 * **주요 창(Window)의 역할**:
   * **하이어라키(Hierarchy)**: 현재 씬에 존재하는 모든 게임 오브젝트를 나열하며, 부모-자식 관계를 관리합니다.
   * **씬(Scene)**: 게임 오브젝트를 시각적으로 배치하고 편집하는 3D 작업 공간입니다.
   * **게임(Game)**: 플레이어가 실제로 보게 될 카메라 시점의 화면입니다.
   * **인스펙터(Inspector)**: 선택한 오브젝트의 속성과 컴포넌트 설정을 변경하는 창입니다.
   * **프로젝트(Project)**: 게임에서 사용하는 모든 에셋(스크립트, 모델, 사운드 등)을 관리하는 창고입니다.
 * **씬 기즈모(Scene Gizmo)**: 씬 카메라의 방향을 표시하며, 클릭 시 해당 축 방향(원근/등각)으로 시점을 즉시 전환합니다.
 * **플레이 버튼**: 게임을 시작(Play), 일시정지(Pause), 한 프레임씩 진행(Step)하는 기능을 제공합니다.
### 2. 유니티 엔진 동작 원리
 * **컴포넌트 패턴**: 상속 대신 여러 기능을 가진 부모를 조립하여 오브젝트를 만드는 방식입니다. 유니티는 유연한 재사용성을 위해 이 방식을 채택하고 있습니다.
 * **MonoBehaviour**: 유니티에서 작성하는 모든 스크립트가 상속받아야 하는 기본 클래스로, 엔진의 제어를 받게 해줍니다.
 * **메시지와 브로드캐스팅**: 유니티는 특정 기능을 실행할 때 메시지를 보내며, 해당 기능을 가진 컴포넌트만 응답하여 실행하는 방식을 사용합니다.
### 3. C# 프로그래밍 기초
 * **변수와 메서드**: 변수는 데이터를 저장하는 장소이며, 메서드는 특정 처리를 수행하는 코드 묶음입니다.
 * **접근 제어자**: public으로 선언된 변수는 유니티 인스펙터 창에 노출되어 직접 값을 수정할 수 있습니다.
 * **생명주기 메서드**:
   * Start(): 스크립트가 활성화될 때 처음 한 번만 실행됩니다.
   * Update(): 매 프레임마다 반복 실행되며, 게임 로직이나 입력을 처리합니다.
 * **컴포넌트 참조**: GetComponent<T>() 함수를 사용하여 동일한 오브젝트에 붙어 있는 다른 컴포넌트를 가져와 제어할 수 있습니다.
### 4. 물리 엔진 및 오브젝트 제어
 * **리지드바디(Rigidbody)**: 오브젝트에 중력과 물리적인 힘을 적용하기 위해 반드시 추가해야 하는 컴포넌트입니다.
 * **충돌 처리(Collision)**:
   * **Collision**: 물리적인 충돌이 발생할 때 사용하며, OnCollisionEnter 메서드로 감지합니다.
   * **Trigger**: 물리적 반작용 없이 겹침만 감지할 때 사용하며, Is Trigger 옵션을 체크해야 합니다.
 * **입력 매니저(Input Manager)**: Input.GetAxis("Horizontal")과 같이 설정된 축 이름을 사용하여 키보드나 게임 패드의 입력을 숫자로 받습니다.
 * **태그(Tag)**: 여러 오브젝트를 논리적 그룹으로 묶어 식별할 때 사용합니다 (예: 벽에 "WALL" 태그 부여).
### 5. 게임 로직 및 UI/이펙트
 * **프리팹(Prefab)**: 복제가 가능하도록 미리 만들어둔 게임 오브젝트의 원본 틀입니다.
   * Instantiate(): 실시간으로 프리팹을 복제하여 생성하는 함수입니다.
   * Destroy(): 게임 오브젝트를 제거하는 함수입니다.
 * **UI 시스템**: Canvas 컴포넌트가 모든 UI 요소의 루트 역할을 하며, UnityEngine.UI 네임스페이스를 사용해야 합니다.
   * **앵커(Anchor)**: 화면 해상도가 변해도 UI 요소의 위치와 크기가 유지되도록 하는 기준점입니다.
 * **게임 매니저**: 점수, 게임오버 상태 등 게임의 전체적인 흐름을 관리하는 중심 스크립트입니다.
 * **이펙트**: Light 컴포넌트로 빛의 범위(Range)와 색상을 조절하거나, 파티클 시스템을 사용하여 특수 효과를 구현합니다.
### 6. 벡터 수학 및 공간/회전
 * **벡터(Vector)**: 방향과 크기를 동시에 가지는 값입니다.
   * magnitude: 벡터의 길이를 나타냅니다.
   * normalized: 방향은 같고 크기가 1인 단위 벡터로 변환합니다.
 * **좌표계**:
   * **전역 공간(Global Space)**: 세상의 절대적 원점(0,0,0)을 기준으로 한 좌표계입니다.
   * **지역 공간(Local Space)**: 부모 오브젝트를 기준으로 한 상대적 좌표계입니다.
 * **회전(Rotation)**:
   * **오일러각(Euler Angles)**: X, Y, Z축을 기준으로 회전하는 방식으로 직관적이지만 '짐벌락(Gimbal Lock)' 문제가 발생할 수 있습니다.
   * **쿼터니언(Quaternion)**: 4차원 수를 이용한 회전 방식으로 짐벌락 문제가 없으며 유니티 내부 회전 처리에 사용됩니다.
